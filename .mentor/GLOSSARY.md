# 📖 Glosarium — Istilah-Istilah Penting

> Kumpulan istilah teknis yang ditemui selama belajar.
> Akan di-update seiring pembelajaran berjalan.

---

## A

| Istilah | Penjelasan |
|---------|-----------|
| **API** (Application Programming Interface) | "Pelayan restoran" — perantara yang menghubungkan frontend (pengunjung) dengan backend (dapur). Frontend minta data, API yang mengambilkan. |
| **App Router** | Sistem routing bawaan Next.js yang menggunakan folder `app/`. Setiap folder = 1 route/halaman. |

## C

| Istilah | Penjelasan |
|---------|-----------|
| **Component** | Potongan UI yang bisa dipakai ulang. Seperti LEGO — bisa disusun untuk membangun halaman. |
| **CRUD** | Create, Read, Update, Delete — 4 operasi dasar untuk mengelola data. |
| **CSS Modules** | Cara styling bawaan Next.js di mana setiap file CSS hanya berlaku untuk 1 component. |

## T (tambahan)

| Istilah | Penjelasan |
|---------|-----------|
| **Tailwind CSS** | Framework CSS berbasis "utility class". Alih-alih menulis CSS terpisah, kamu langsung tulis class di HTML seperti `className="flex p-4 text-lg bg-blue-500"`. Seperti LEGO warna — tinggal tempel class untuk styling. |

## D

| Istilah | Penjelasan |
|---------|-----------|
| **Dynamic Route** | Route yang bisa berubah-ubah, contoh: `/projects/1`, `/projects/2`. Menggunakan `[slug]` di nama folder. |

## F

| Istilah | Penjelasan |
|---------|-----------|
| **Framework** | "Kerangka kerja" — seperti template rumah yang sudah ada strukturnya, tinggal kita isi dan dekorasi. Next.js adalah framework untuk React. |
| **Fullstack** | Mengerjakan frontend (tampilan) DAN backend (server/database) sekaligus. |

## J

| Istilah | Penjelasan |
|---------|-----------|
| **JSX/TSX** | Cara menulis HTML di dalam JavaScript/TypeScript. JSX untuk JS, TSX untuk TS. |

## L

| Istilah | Penjelasan |
|---------|-----------|
| **Layout** | Template yang membungkus halaman. Contoh: navbar dan footer yang muncul di semua halaman. |

## M

| Istilah | Penjelasan |
|---------|-----------|
| **Metadata** | Informasi tentang halaman (judul, deskripsi) yang dipakai oleh Google dan social media. |
| **Middleware** | Code yang berjalan SEBELUM request sampai ke halaman. Biasa dipakai untuk cek login. |

## N

| Istilah | Penjelasan |
|---------|-----------|
| **Next.js** | Framework React untuk membuat website fullstack. Dibuat oleh Vercel. |

## P

| Istilah | Penjelasan |
|---------|-----------|
| **Props** | "Properties" — cara mengirim data dari component induk ke component anak. Seperti memberi instruksi ke component. |

## R

| Istilah | Penjelasan |
|---------|-----------|
| **Route** | URL path di website. Contoh: `/about` adalah route untuk halaman About. |
| **React** | Library JavaScript untuk membuat UI. Next.js dibangun di atas React. |

## S

| Istilah | Penjelasan |
|---------|-----------|
| **Server Action** | Function yang berjalan di server, biasa dipakai untuk handle form submission. |
| **Server Component** | Component yang di-render di server (bukan di browser). Default di Next.js App Router. |
| **SSR** (Server-Side Rendering) | Halaman di-render di server lalu dikirim ke browser. Bagus untuk SEO. |

## T

| Istilah | Penjelasan |
|---------|-----------|
| **TypeScript** | JavaScript dengan tambahan "tipe data". Membantu mendeteksi error sebelum code dijalankan. |

## V

| Istilah | Penjelasan |
|---------|-----------|
| **Vercel** | Platform untuk deploy (meng-online-kan) website Next.js. Gratis untuk project kecil. |

---

> 💡 **Tip:** Istilah baru akan ditambahkan setiap kali muncul dalam pembelajaran.
